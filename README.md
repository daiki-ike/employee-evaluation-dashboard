# 社員評価ダッシュボード

Google スプレッドシートからデータを読み込んで、社員の売上・利益と評価を可視化するWebアプリケーションです。

## 機能

### 1. 認証機能
- **部長アカウント（6種類）**: 自部署のメンバーのみ閲覧可能
- **社長アカウント**: 全社員のデータを閲覧可能
- **管理者アカウント**: 全社員のデータ閲覧 + データアップロード権限

### 2. 売上・利益ダッシュボード
- 全体・部署別・個人別の売上と利益を表示
- 過去2年分のデータを年度別に可視化
- 棒グラフで年度推移を表示
- フィルター機能（年度、部署、社員名で絞り込み）

### 3. 評価シート
- 自己評価と部長評価を比較
- 60個の設問ごとに評価の乖離を可視化
- 配点を考慮した得点計算
- 乖離が大きい設問をハイライト表示

## セットアップ

### 必要要件
- Node.js 20以上
- npm

### インストール

```bash
cd employee-evaluation-dashboard
npm install
```

### ローカルでの起動

```bash
npm run dev
```

ブラウザで `http://localhost:5173` にアクセスしてください。

## データ構造

### 売上・利益データ
Google スプレッドシートの構造：

| 氏名 | 部署 | 年度 | 売上 | 利益 |
|------|------|------|------|------|
| 山田太郎 | 営業1部 | 2023 | 10000000 | 2000000 |
| 山田太郎 | 営業1部 | 2024 | 12000000 | 2500000 |

### 評価データ
以下の列を含むシート：
- A列: No.（設問番号）
- B列: 大カテゴリ
- C列: 小カテゴリ
- D列: 審査内容
- E列: 自己評価
- F列: 部長評価

### 配点データ
以下の列を含むシート：
- A列: 日付
- B列: 氏名
- C列: 部署
- D列: 役職
- E列以降: 各設問の配点

## デプロイ

### GitHub Pagesへのデプロイ

1. GitHubリポジトリを作成

2. コードをプッシュ
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin <your-repo-url>
git push -u origin main
```

3. GitHub リポジトリの Settings > Pages で以下を設定：
   - Source: GitHub Actions

4. mainブランチにプッシュすると自動的にデプロイされます

デプロイ後、`https://<username>.github.io/employee-evaluation-dashboard/` でアクセスできます。

## Google スプレッドシートの設定

1. スプレッドシートを「リンクを知っている全員が閲覧可能」に設定
2. 管理者アカウントでログイン
3. 「データアップロード」ページで各スプレッドシートのURLとシート名を入力
4. データを読み込み
